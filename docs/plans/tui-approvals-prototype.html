克制要点

相比常规版砍掉了:legend/图例块、每条的双行排版、独立 warnings 段,以及第二条分隔线(只留主从那条竖线)。特例全压成行内短记号——sudo 禁 host = scope 芯片里把 h 灰显成括号 o/s(h) 并配 ·sudo;管道精确 = ·pipe;命令超长用 … 截断,完整命令只在右侧选中详情里给。仍然清楚的原因:每行的 o/s/h 芯片与底部 [o][s][h][d] 键位一一对应,不需文字解释就知道这条能按哪几个键;顶部一句 4 pending、底部一句 decide >1 ap_7f3a,操作者无需扫视即知「现在决定哪条、按什么」。

克制要点

相比常规版砍掉了:会话分组与意图标签列(fix 502/backup)、表头、legend/说明块、每条的多行展开、范围整句解释——整屏只剩一处分隔线。约束信息无损保留:列位即语义(候选 prefix/exact、范围 o/s/h),特例用极短记号——·sudo 配灰显 ·h(host 禁用,无括号即不可按)、·pipe 配 exact(只精确);▸ 行光标 + 底部单条键位让「决定哪条·按什么」一眼即明,空态收敛为一行「无待批 · 全部已处理」。
