AgentSSH TUI · Audit redesign draft

Audit 默认看 session，进入后只看命令结果。

这个初稿刻意把视觉层级拆开：全局状态是 rail，会话是 review queue，命令是 session 内的主信息，record/hash 只作为底层审计脚注，不再抢占视线。

不再分栏主列表和详情分别占满一屏，减少“左边选什么、右边看什么”的认知负担。
不再展示 recordstarted/completed/denied 是审计账本事件，不是用户要操作的对象。

底层审计：每条命令仍由 append-only records 组成，用于哈希链和崩溃恢复；TUI 默认只展示投影后的 command result。需要查 seq/hash 时走 verify/debug，而不是主阅读路径。
层级 1 · rail只放产品位置、总数、链状态、快捷键。它是全局状态，不和业务行混排。
层级 2 · queue默认 session 列表按更新时间倒序；异常只做行内状态提示，不改变排序规则。
层级 3 · commandssession 详情只列命令结果；被拒绝、失败、成功、运行中有不同颜色和文案。
层级 4 · evidencerecord/hash 保留为底层能力，但只在 verify/debug 里出现，避免污染主 UI。
